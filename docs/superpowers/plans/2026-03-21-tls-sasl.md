**Goal:** Enable go-bench to connect to TLS-encrypted Kafka/Redpanda clusters and authenticate via SASL (PLAIN, SCRAM-SHA-256, SCRAM-SHA-512).

**Architecture:** A new `internal/tlsconfig` package builds a `*tls.Config` from CLI flags; a `saslOpts` helper in bench.go returns the correct franz-go SASL option. **Tech Stack:** `crypto/tls`, `crypto/x509` (stdlib); `github.com/twmb/franz-go/pkg/kgo` (`DialTLSConfig`, `SASL`); `github.com/twmb/franz-go/pkg/sasl/plain` (`plain.Auth.AsMechanism()`); `github.com/twmb/franz-go/pkg/sasl/scram` (`scram.Auth.AsSha256Mechanism()`, `scram.Auth.AsSha512Mechanism()`).

### Background

This is a standalone package — no dependencies on other internal packages. It wraps stdlib `crypto/tls` and `crypto/x509` into a single `Build` function. The package is named `tlsconfig` (not `tls`) to avoid an import name collision with stdlib `crypto/tls` in any file that imports both.

Three behaviours based on the two inputs:
- `enabled=false, caCertPath=""` → return `nil, nil` (TLS disabled)
- `enabled=true, caCertPath=""` → return `&tls.Config{}, nil` (system root CAs)
- `caCertPath != ""` → load PEM from file, build cert pool, return `&tls.Config{RootCAs: pool}, nil`; a non-empty `caCertPath` implies TLS regardless of the `enabled` flag

```go
		SerialNumber: big.NewInt(1),
		Subject:      pkix.Name{CommonName: "test-ca"},

		NotAfter:     time.Now().Add(time.Hour),
```

```go
// Package tlsconfig builds *tls.Config values for use with franz-go clients.
// Named tlsconfig (not tls) to avoid import-name collision with crypto/tls.
package tlsconfig

import (
	"crypto/tls"
	"crypto/x509"
	"fmt"
	"os"
)

// Build returns a *tls.Config for use with franz-go's kgo.DialTLSConfig option.
//
//   - enabled=false, caCertPath="" → nil, nil  (TLS disabled; caller omits the option)
//   - enabled=true,  caCertPath="" → &tls.Config{} using system root CAs
//   - caCertPath non-empty         → &tls.Config{RootCAs: pool} loaded from the PEM file;
//                                    implies enabled regardless of the enabled flag
//
// Returns a non-nil error if caCertPath is set but the file cannot be read or
// contains no valid PEM certificates.
func Build(enabled bool, caCertPath string) (*tls.Config, error) {
	if !enabled && caCertPath == "" {
		return nil, nil
	}
	if caCertPath == "" {
		return &tls.Config{}, nil
	}
	pemBytes, err := os.ReadFile(caCertPath)
	if err != nil {
		return nil, fmt.Errorf("read CA cert %q: %w", caCertPath, err)
	}
	pool := x509.NewCertPool()
	if !pool.AppendCertsFromPEM(pemBytes) {
		return nil, fmt.Errorf("no valid PEM certificates found in %q", caCertPath)
	}
	return &tls.Config{RootCAs: pool}, nil
}
```

## Task 3: Add `extraOpts` to topic.Create and topic.Delete

### Background

`internal/topic/topic.go` has two exported functions:

```go

func Delete(ctx context.Context, brokers []string, name string) error
```

Each creates its own `kgo.Client` with only `kgo.SeedBrokers`. Against a TLS or SASL-required cluster these calls will fail. Adding a variadic `extraOpts ...kgo.Opt` parameter to both is backward-compatible — existing callers (including tests) that pass no extra opts continue to compile and behave identically.

No new tests are required — the function logic is unchanged. The variadic change is structural. The existing `TestValidateName` test doesn't call `Create` or `Delete` directly and requires no modification.

```go
// saslOpts returns the franz-go SASL option for the configured mechanism.
// Returns nil when SASLMechanism is empty (no authentication); nil spreads
// cleanly with ... in append so no nil-guard is needed at call sites.
func saslOpts(cfg *config.Config) []kgo.Opt {
	switch cfg.SASLMechanism {
	case "plain":
		return []kgo.Opt{kgo.SASL(plain.Auth{
			User: cfg.SASLUsername,
			Pass: cfg.SASLPassword,
		}.AsMechanism())}
	case "scram-sha-256":
		return []kgo.Opt{kgo.SASL(scram.Auth{
			User: cfg.SASLUsername,
			Pass: cfg.SASLPassword,
		}.AsSha256Mechanism())}
	case "scram-sha-512":
		return []kgo.Opt{kgo.SASL(scram.Auth{
			User: cfg.SASLUsername,
			Pass: cfg.SASLPassword,
		}.AsSha512Mechanism())}
	}
	return nil
}
```
